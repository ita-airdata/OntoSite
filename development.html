Guia de Desenvolvimento

Orientações e referências para contribuidores da ontologia AirData.

Padrões de Qualidade (ROBOT)

Para garantir a consistência semântica e estrutural, todas as contribuições devem aderir aos seguintes critérios obrigatórios:

  1. Labels Únicos em Português: Use apenas um rdfs:label com a tag de idioma @pt-br por classe. Traduções para o inglês devem ser movidas para a anotação skos:prefLabel.
  2. Definições Formais: Toda classe deve possuir uma definição em texto natural utilizando a propriedade iao:IAO_0000115 (também em português).
  3. Metadados da Ontologia: O cabeçalho da ontologia deve conter propriedades descritivas como dcterms:description e informações de licença via dcterms:license.
  4. Hierarquia Explícita: Classes de nível superior (sem superclasse lógica no domínio) devem declarar explicitamente rdfs:subClassOf owl:Thing.

Ferramentas e Referências

Abaixo estão listadas as especificações e ferramentas essenciais utilizadas no desenvolvimento da ontologia:

Linguagem e Ferramentas

  • OWL (Web Ontology Language): Especificação oficial do W3C para a linguagem de ontologias.
  • ROBOT: Ferramenta de linha de comando para automação de tarefas de ontologia, utilizada para validação e geração de relatórios de qualidade.

Modelos de Referência (Domínio Aeronáutico)

  • AIXM (Aeronautical Information Exchange Model): Referência principal para modelagem de dados de aviação. Disponível também a documentação HTML da versão 5.1.1.
  • FIXM (Flight Information Exchange Model): Padrão global para intercâmbio de informações de voo.
  • SWIM (System Wide Information Management): Referência de interoperabilidade.
